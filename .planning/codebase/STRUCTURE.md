## Directory Purposes

**app/:**
- Purpose: All backend application code (models, controllers, business logic)
- Contains: PHP classes organized by responsibility

**app/Http/Controllers/:**
- Purpose: HTTP request handlers that coordinate models and views
- Controllers inherit from base `Controller`
- Settings controllers in `Settings/` subdirectory handle profile, password, 2FA
- Pattern: Method returns `Inertia::render()` for pages or `RedirectResponse` for mutations

**app/Http/Requests/:**
- Purpose: Form validation with optional authorization
- File naming: `[Entity][Action]Request.php` (e.g., `ProfileUpdateRequest.php`)
- Contains array method `rules()` returning validation rules
- Can use trait-based rule sharing via `Concerns/`

**app/Models/:**
- Purpose: Eloquent model definitions representing database tables
- Only `User.php` currently; extended by Fortify traits
- Uses constructor property promotion, explicit casts

**app/Actions/:**
- Purpose: Encapsulated business logic separate from HTTP controllers
- `Fortify/` contains actions implementing Fortify contracts
- Example: `CreateNewUser` implements `CreatesNewUsers` contract

**app/Providers/:**
- Purpose: Service provider bootstrapping
- `AppServiceProvider` configures defaults (Date handling, password rules, etc)
- `FortifyServiceProvider` registers Fortify actions

**app/Concerns/:**
- Purpose: Reusable traits for shared functionality
- Contains validation rule methods used by FormRequests and Actions
- Example: `ProfileValidationRules`, `PasswordValidationRules`

**bootstrap/app.php:**
- Purpose: Application bootstrap configuration
- Defines routing files, middleware stack, exception handling
- Uses Laravel 12 fluent API: `Application::configure()`
- Middleware registered via `withMiddleware()` callback

**config/:**
- Purpose: Centralized configuration files
- `inertia.php` - SSR setup, page path configuration
- `fortify.php` - Which authentication features are enabled
- Configuration accessed via `config()` function (never `env()` in code)

**database/migrations/:**
- Purpose: Incremental schema changes
- Naming: Timestamp + description (e.g., `0001_01_01_000000_create_users_table.php`)
- Run via `php artisan migrate`

**database/factories/:**
- Purpose: Model factory classes for test data generation
- `UserFactory` creates test users with customizable states
- Used in tests via `User::factory()->create()`

**resources/js/pages/:**
- Purpose: Vue 3 components that map to URL routes
- Naming: TitleCase matching the route path structure
- Nested in subdirectories matching URL structure
- Example: `/settings/profile` → `resources/js/pages/settings/Profile.vue`

**resources/js/components/:**
- Purpose: Reusable Vue 3 components (not pages)
- `ui/` subdirectory contains Shadcn/ui components
- `AppSidebarHeader.vue` and other custom components
- Components are imported and used by pages/layouts

**resources/js/layouts/:**
- Purpose: Page wrapper components for consistent UI structure
- `AppLayout.vue` - wraps authenticated app pages
- `AuthLayout.vue` - wraps auth pages (login, register)
- `SettingsLayout.vue` - wraps settings pages
- Layouts receive props like breadcrumbs and slot content

**resources/js/composables/:**
- Purpose: Reusable Vue 3 logic and state management
- `useAppearance.ts` - theme/dark mode state
- `useTwoFactorAuth.ts` - 2FA logic
- Pattern: Export function returning reactive state and methods

**resources/js/types/:**
- Purpose: TypeScript type definitions
- `index.ts` - central export point
- `auth.ts` - authentication-related types
- `navigation.ts` - sidebar/navigation types
- Imported and used throughout Vue components

**resources/js/actions/ (Generated):**
- Purpose: Wayfinder-generated type-safe route/action functions
- Auto-generated from Laravel controllers via `@laravel/vite-plugin-wayfinder`
- Mirrors backend PHP namespace structure
- Import pattern: `import StoreUserController from '@/actions/App/Http/Controllers/StoreUserController'`

**resources/js/routes/ (Generated):**
- Purpose: Wayfinder-generated named route functions
- Auto-generated from `Route::name()` calls in Laravel
- Import pattern: `import { dashboard } from '@/routes'`
- Supports parameter binding and query options

**routes/web.php:**
- Purpose: Web application route definitions
- Routes return `Inertia::render()` for pages
- Middleware applied via `middleware()` chain
- Includes `routes/settings.php`

**routes/settings.php:**
- Purpose: Settings-related routes
- Routes grouped with `auth` middleware
- Further routes guarded with `verified` middleware for 2FA
- Controllers in `App\Http\Controllers\Settings\`

**tests/Feature/:**
- Purpose: Integration/feature tests (HTTP request level)
- Test actual routes and controller logic
- Pest framework with `test()` function syntax
- Database refreshed per test via `RefreshDatabase` trait

**Key File Locations**

**Entry Points:**
- `public/index.php`: Web server entry point for all HTTP requests
- `resources/js/app.ts`: Frontend app initialization
- `bootstrap/app.php`: Application configuration bootstrap

**Configuration:**
- `config/fortify.php`: Which authentication features (login, register, 2FA, etc) are enabled
- `config/inertia.php`: SSR and component discovery settings
- `.env`: Environment variables (never read directly in code)

**Core Logic:**
- `app/Http/Controllers/Settings/`: Settings page logic
- `app/Actions/Fortify/`: Authentication action logic
- `app/Concerns/`: Shared validation rule logic

**Testing:**
- `tests/Feature/Auth/`: Authentication tests
- `tests/Feature/Settings/`: Settings feature tests
- `tests/Pest.php`: Pest configuration and helpers
- `tests/TestCase.php`: Base test case with Fortify helpers

**Frontend Components:**
- `resources/js/pages/`: Page components mapped to routes
- `resources/js/layouts/`: Layout wrappers
- `resources/js/composables/`: Reusable Vue logic

## Naming Conventions

**Files:**
- Controllers: `[Entity][Action]Controller.php` (e.g., `ProfileController.php`)
- Models: `[EntityName].php` in singular form (e.g., `User.php`)
- Migrations: `YYYY_MM_DD_HHMMSS_[description].php`
- Form Requests: `[Entity][Action]Request.php` (e.g., `ProfileUpdateRequest.php`)
- Vue Pages: `[PascalCase].vue` matching route hierarchy (e.g., `Dashboard.vue`, `settings/Profile.vue`)
- Vue Components: `[PascalCase].vue` (e.g., `AppSidebarHeader.vue`)
- Composables: `use[Feature].ts` in camelCase (e.g., `useAppearance.ts`)
- Types: `[name].ts` (e.g., `auth.ts`, `navigation.ts`)
- Tests: `[Feature]Test.php` (e.g., `ProfileUpdateTest.php`)

**Directories:**
- Controllers: `[Entity]/` for grouped controllers (e.g., `Settings/`)
- Routes: Lowercase with hyphens (e.g., `/settings/two-factor`)
- Frontend components: Lowercase with hyphens in URLs, PascalCase in files
- Tests: Mirror backend structure (e.g., `tests/Feature/Settings/`)

## Where to Add New Code

**New Page/Feature:**
1. Create route in `routes/web.php` or `routes/settings.php`: `Route::inertia('path', 'ComponentName')`
2. Create page component: `resources/js/pages/[Path]/[ComponentName].vue`
3. If needs backend data: Create controller method returning `Inertia::render()`
4. Create test in `tests/Feature/[FeatureName]Test.php`

**New Settings Feature:**
1. Create controller in `app/Http/Controllers/Settings/[Feature]Controller.php`
2. Create Form Request in `app/Http/Requests/Settings/[Feature]Request.php`
3. Add routes in `routes/settings.php` grouped with appropriate middleware
4. Create page component in `resources/js/pages/settings/[Feature].vue`
5. Create layout in `resources/js/layouts/settings/[Feature]Layout.vue` (if complex)
6. Create test in `tests/Feature/Settings/[Feature]Test.php`

**New Component/Module:**
- Reusable Vue component: `resources/js/components/[Name].vue`
- Composable logic: `resources/js/composables/use[Feature].ts`
- Type definitions: `resources/js/types/[name].ts`
- Shared validation rules: `app/Concerns/[Name]ValidationRules.php` (trait)

**New Authorization/Business Logic:**
- Complex action: Create `app/Actions/[Domain]/[Action].php` implementing appropriate contract
- Shared validation: Add trait to `app/Concerns/[Name]ValidationRules.php`
- Model logic: Add methods to `app/Models/[Model].php` (relationships, accessors, etc)

**Utilities:**
- Backend helpers: `app/Helpers/` (create if needed) or utility classes in `app/`
- Frontend utilities: `resources/js/lib/` directory
- Both should be organized by domain/feature

## Special Directories

**storage/logs/:**
- Purpose: Application logs written by Laravel logging system
- Generated: Yes (created at runtime)
- Committed: No (ignored in .gitignore)

**public/build/:**
- Purpose: Compiled frontend assets (CSS, JS) from Vite build
- Generated: Yes (by `npm run build`)
- Committed: No (should be in .gitignore)

**database/database.sqlite:**
- Purpose: SQLite development database
- Generated: Yes (created by migrations)
- Committed: No (local development only)

**bootstrap/cache/:**
- Purpose: Laravel's internal configuration cache
- Generated: Yes (by `php artisan config:cache`)
- Committed: No (regenerated automatically)

**vendor/ and node_modules/:**
- Purpose: Package dependencies
- Generated: Yes (by composer/npm install)
- Committed: No (lock files committed instead)
