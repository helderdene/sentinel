  <action>
    **1. Create migration** via `php artisan make:migration add_decommissioned_at_to_units_table --no-interaction`:
    - Add nullable `decommissioned_at` timestamp column to units table
    - This separates admin lifecycle (decommission) from operational status (dispatch workflow)

    **2. Update Unit model** (app/Models/Unit.php):
    - Add `decommissioned_at` to $fillable array
    - Add `'decommissioned_at' => 'datetime'` to casts() method
    - Add `scopeActive(Builder $query): Builder` that filters `whereNull('decommissioned_at')` -- used by dispatch queries
    - Add `scopeCommissioned(Builder $query): Builder` as alias for clarity

    **3. Create AdminUnitController** via `php artisan make:controller Admin/AdminUnitController --resource --no-interaction`:
    - `index()`: Query `Unit::query()->withCount('users')->with('users:id,name,unit_id')->orderBy('type')->orderBy('id')->get()`. Also pass `User::where('role', UserRole::Responder)->select('id', 'name', 'unit_id')->orderBy('name')->get()` as `responders` prop for crew assignment. Pass `UnitType::cases()` and `[UnitStatus::Available, UnitStatus::Offline]` as `types` and `statuses` props. Render Inertia 'admin/Units'.
    - `create()`: Same supporting data (responders, types, statuses). Render 'admin/UnitForm'.
    - `store(StoreUnitRequest)`: Auto-generate unit ID using type prefix mapping (AMB/FIRE/RESCUE/POLICE/BOAT) + MAX sequence from existing units of that type + 1, zero-padded to 2 digits. Use PostgreSQL regex: `MAX(CAST(SUBSTRING(id FROM '[0-9]+$') AS INTEGER))`. Auto-generate callsign as default (e.g. "Ambulance 3") but allow override if callsign is provided in request. Sync crew_ids via two-step User update. Redirect to index with success flash.
    - `edit(Unit $unit)`: Load unit with users. Same supporting data. Render 'admin/UnitForm'.
    - `update(UpdateUnitRequest, Unit $unit)`: Update unit fields. Sync crew_ids bidirectionally: (1) `User::where('unit_id', $unit->id)->whereNotIn('id', $crewIds)->update(['unit_id' => null])`, (2) `User::whereIn('id', $crewIds)->update(['unit_id' => $unit->id])`. Redirect with success.
    - `destroy(Unit $unit)`: Check `$unit->activeIncidents()->count()` -- if > 0, redirect back with error "Cannot decommission unit with active incidents." Otherwise set `decommissioned_at = now()`, unassign all crew `$unit->users()->update(['unit_id' => null])`. Redirect with success.
    - `recommission(Unit $unit)`: Clear `decommissioned_at`, set `status = UnitStatus::Available`. Redirect with success.

    **4. Create StoreUnitRequest** via `php artisan make:request Admin/StoreUnitRequest --no-interaction`:
    - Rules (array-style): type required + Rule::in(UnitType values), callsign nullable string max:50, agency required string max:50, crew_capacity required integer min:1 max:20, status required + Rule::in(['AVAILABLE', 'OFFLINE']), shift nullable + Rule::in(['day', 'night']), notes nullable string max:1000, crew_ids nullable array, crew_ids.* exists:users,id

    **5. Create UpdateUnitRequest** via `php artisan make:request Admin/UpdateUnitRequest --no-interaction`:
    - Same rules as Store except type is excluded (unit ID/type cannot change after creation)

    **6. Register routes** in routes/admin.php:
    - Add `use App\Http\Controllers\Admin\AdminUnitController;` import
    - Add `Route::resource('units', AdminUnitController::class);`
    - Add `Route::post('units/{unit}/recommission', [AdminUnitController::class, 'recommission'])->name('units.recommission');`

    **7. Update routes/web.php:**
    - Replace the ComingSoon placeholder at line 97-100 with a redirect: `Route::redirect('units', '/admin/units')->name('units.index');`
    - This preserves backward compatibility for bookmarks while routing to the new admin page
