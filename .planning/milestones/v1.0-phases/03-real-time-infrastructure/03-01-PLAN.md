  <action>
    **Step 1: Create 6 broadcast event classes** using `php artisan make:event`. Each event implements `ShouldBroadcast` and `ShouldDispatchAfterCommit`, uses `Dispatchable`, `InteractsWithSockets`, `SerializesModels` traits, and defines explicit `broadcastWith()` payloads.

    **IncidentCreated** — broadcasts on `PrivateChannel('dispatch.incidents')`:
    - Constructor: `public Incident $incident`
    - broadcastWith: `id`, `incident_no`, `priority` (->value), `status` (->value), `incident_type` ($this->incident->incidentType?->name), `location_text`, `barangay` ($this->incident->barangay?->name), `channel` (->value), `created_at` (->toISOString())

    **IncidentStatusChanged** — broadcasts on `PrivateChannel('dispatch.incidents')`:
    - Constructor: `public Incident $incident, public IncidentStatus $oldStatus`
    - broadcastWith: `id`, `incident_no`, `old_status` ($this->oldStatus->value), `new_status` ($this->incident->status->value), `priority` (->value)

    **UnitLocationUpdated** — broadcasts on `PrivateChannel('dispatch.units')`:
    - Constructor: `public Unit $unit`
    - broadcastWith: `id`, `callsign`, `latitude` (extract from coordinates), `longitude` (extract from coordinates), `updated_at` (->toISOString())

    **UnitStatusChanged** — broadcasts on `PrivateChannel('dispatch.units')`:
    - Constructor: `public Unit $unit, public UnitStatus $oldStatus`
    - broadcastWith: `id`, `callsign`, `old_status` ($this->oldStatus->value), `new_status` ($this->unit->status->value)

    **AssignmentPushed** — broadcasts on `PrivateChannel('user.' . $this->userId)`:
    - Constructor: `public Incident $incident, public string $unitId, public int $userId`
    - broadcastWith: full incident data array (id, incident_no, priority, status, incident_type, location_text, barangay, coordinates, notes) + unit_id

    **MessageSent** — broadcasts on `PrivateChannel('user.' . $this->recipientId)`:
    - Constructor: `public int $recipientId, public string $incidentId, public int $senderId, public string $senderName, public string $body`
    - broadcastWith: `incident_id`, `sender_id`, `sender_name`, `body`, `sent_at` (now()->toISOString())

    For UnitLocationUpdated, extract lat/lng from the Magellan Point geometry. Check how the Unit model stores coordinates — it uses Magellan geography Point cast. In broadcastWith(), access `$this->unit->coordinates` which is a Magellan Point object, then extract latitude/longitude via `$this->unit->coordinates->latitude` and `$this->unit->coordinates->longitude`. Handle null coordinates gracefully.

    **Step 2: Define channel authorization** in `routes/channels.php`:

    4 channels per user decisions:
    - `dispatch.incidents` — private, authorized for dispatcher/supervisor/admin roles
    - `dispatch.units` — private, authorized for dispatcher/supervisor/admin roles
    - `user.{id}` — private, authorized only when user id matches (cast $id to int for comparison since User uses auto-increment int PK)
    - `presence-dispatch` — presence channel, authorized for dispatcher/supervisor/admin, returns `['id' => $user->id, 'name' => $user->name, 'role' => $user->role->value]`

    Import `App\Enums\UserRole`, `App\Models\User`, and `Illuminate\Support\Facades\Broadcast`.

    **Step 3: Create StateSyncController** as an invokable controller:
    Run `php artisan make:controller StateSyncController --invokable --no-interaction`.

    The `__invoke()` method returns a JsonResponse with:
    - `incidents`: All PENDING incidents with incidentType + barangay eager-loaded, ordered by priority (P1 first) then created_at asc
    - `channelCounts`: Pending incidents grouped by channel with count
    - `units`: All non-OFFLINE units with id, callsign, type, status, coordinates

    Use the exact same query pattern as IncidentController::queue() for incidents. For units, select only needed columns.

    **Step 4: Register state-sync route** in `routes/web.php`:
    Add `Route::get('/state-sync', StateSyncController::class)->name('state-sync')` inside the authenticated + role middleware group (dispatcher, supervisor, admin). Place it in the existing auth middleware group alongside the incident routes.

    **Step 5: Wire IncidentCreated event dispatch** into `IncidentController::store()`:
    After the incident is created (after `Incident::query()->create($data)`), load the incidentType and barangay relations, then dispatch the event:
    ```php
    $incident->load('incidentType', 'barangay');
    IncidentCreated::dispatch($incident);
    ```
    Add the import: `use App\Events\IncidentCreated;`

    **Step 6: Run Pint** on all modified PHP files: `vendor/bin/pint --dirty --format agent`.
