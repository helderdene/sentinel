<!-- Key token architecture the executor needs to understand -->

From resources/css/app.css — Token cascade:
```
--t-* tokens (authoritative hex values in :root and .dark)
  -> Shadcn variables (--background: var(--t-bg), etc.)
    -> @theme inline (--color-t-bg: var(--t-bg), etc.)
      -> Tailwind utility classes (bg-t-bg, text-t-text, etc.)
```

The executor ONLY needs to change the hex values in :root and .dark blocks.
Shadcn remapping and @theme inline references stay the same (they reference vars, not raw values).

From report-app/src/assets/tokens.css — Separate token file:
Uses @media (prefers-color-scheme: dark) instead of .dark selector.
Has --t-accent-lo token not in main app. Must be updated to Sentinel equivalent.

From report-app/src/assets/app.css — Font registration:
@theme inline block registers --font-mono: 'Space Mono' -> change to 'DM Mono'

From resources/views/app.blade.php — Font loading:
Google Fonts <link> loads DM Sans + Space Mono -> replace with DM Sans + DM Mono + Bebas Neue
Inline <style> sets html background-color for dark mode flash prevention -> update to #05101E
<meta name="theme-color"> -> update to #042C53
