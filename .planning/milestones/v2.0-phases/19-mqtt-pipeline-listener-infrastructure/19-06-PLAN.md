       **§1 Overview** — Phase 19 MQTT listener architecture (listener → TopicRouter → 4 handlers; `fras_events` disk; `mqtt` log channel).

       **§2 Local Dev Prerequisites** — Install Mosquitto:
       - macOS: `brew install mosquitto && brew services start mosquitto`
       - Linux: `sudo apt-get install -y mosquitto mosquitto-clients && sudo systemctl enable --now mosquitto`
       - Verify: `mosquitto_sub -h localhost -t 'mqtt/face/#' -v` in one terminal; `mosquitto_pub -h localhost -t mqtt/face/heartbeat -m '{}'` in another.

       **§3 Production Supervisor Block** — add `/etc/supervisor/conf.d/irms-mqtt.conf`:
       ```ini
       [program:irms-mqtt]
       process_name=%(program_name)s_%(process_num)02d
       command=php /var/www/irms/artisan irms:mqtt-listen --max-time=3600
       autostart=true
       autorestart=unexpected
       stopwaitsecs=30
       stopsignal=TERM
       user=www-data
       numprocs=1
       redirect_stderr=true
       stdout_logfile=/var/log/supervisor/irms-mqtt.log
       stdout_logfile_maxbytes=50MB
       stdout_logfile_backups=10
       ```
       Reload + start:
       ```bash
       sudo supervisorctl reread
       sudo supervisorctl update
       sudo supervisorctl start irms-mqtt:*
       ```
       **Never put this under Horizon** — Pitfall 6 mandates separation. Horizon restart must not affect the listener.

       **§4 Deploy Protocol (Pitfall 7 mitigation)** — Add to the existing deploy script AFTER `supervisorctl restart irms-horizon:*` and AFTER migrations run:
       ```bash
       sudo supervisorctl restart irms-mqtt:*
       ```
       Verify the listener picked up new code:
       ```bash
       tail -n 20 storage/logs/mqtt-$(date +%Y-%m-%d).log | grep "MQTT listener started"
       ```
       Expected: fresh "MQTT listener started" log line with the new deploy timestamp. If missing, the listener did NOT restart — investigate before leaving the deploy.

       **§5 Post-Deploy Smoke Test** — Publish a sentinel message and verify the pipeline:
       ```bash
       # Terminal 1 — watch the mqtt log:
       tail -f storage/logs/mqtt-$(date +%Y-%m-%d).log

       # Terminal 2 — publish a sentinel heartbeat:
       mosquitto_pub -h <MQTT_HOST> -p 1883 -u <MQTT_USERNAME> -P <MQTT_PASSWORD> \
         -t mqtt/face/heartbeat -m '{"facesluiceId":"irms-smoketest"}'
       ```
       Expected in Terminal 1: a log line showing the heartbeat topic was routed (and a "Heartbeat for unknown camera" warning if `irms-smoketest` is not registered — that is the expected smoke test path for Phase 19).

       This section MUST include the literal phrase **"smoke test runbook"** as a header so tooling can grep for it.

       **§6 Troubleshooting** —
       - Listener will not start: check `tail storage/logs/mqtt-*.log` for connection-refused (Mosquitto down or env MQTT_HOST/PORT wrong).
       - Banner stuck on SILENT: run `php artisan schedule:work` in dev to verify watchdog fires; check `redis-cli GET 'mqtt:listener:last_message_received_at'`.
       - Duplicate RecPush showing up multiple times in logs: expected — they log "Duplicate RecPush rejected at DB layer" and return early (D-03 idempotency).

       **§7 Deferred** — reference section linking to Phase 20 (camera admin), Phase 21 (FrasIncidentFactory bridge), Phase 22 (retention purge + signed URLs + DPA audit log).
