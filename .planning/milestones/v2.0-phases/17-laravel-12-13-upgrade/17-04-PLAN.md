Use two-argument signature; bypass the `wherePivotNull` scope with a raw pivot query so post-resolution responders retain access:

    ```php
    Gate::define('download-incident-report', function (User $user, Incident $incident): bool {
        if (in_array($user->role, [
            UserRole::Operator,
            UserRole::Dispatcher,
            UserRole::Supervisor,
            UserRole::Admin,
        ], true)) {
            return true;
        }

        if ($user->role === UserRole::Responder) {
            if ($user->unit_id === null) {
                return false;
            }

            // Raw pivot query: bypasses Incident::assignedUnits()->wherePivotNull('unassigned_at')
            // so the responder who resolved the incident (ResponderController::resolve sets
            // unassigned_at=now() at line 337) retains post-resolution download access.
            return DB::table('incident_unit')
                ->where('incident_id', $incident->id)
                ->where('unit_id', $user->unit_id)
                ->exists();
        }

        return false;
    });
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated browser → Laravel route | Session-cookie holder issues GET to `/incidents/{incident}/report.pdf`. Server must enforce role + per-incident assignment authorization. |
| Laravel controller → local disk | Controller reads file path stored in DB (`report_pdf_url`) and streams from `Storage::disk('local')`. Must not allow path traversal or reading files outside the `incident-reports/` prefix. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-05 | Information Disclosure | `IncidentController::downloadReport` | mitigate | `Gate::authorize('download-incident-report', $incident)` rejects cross-unit responders (responder with unit_id not in `incident_unit` pivot for this incident, regardless of `unassigned_at`). Pest Test 6 + Test 7 in DownloadReportTest.php are the regression oracle. |
| T-17-06 | Information Disclosure | File path resolution | mitigate | `report_pdf_url` is written ONLY by `GenerateIncidentReport::handle()` at a deterministic path `incident-reports/{incident_no}.pdf` — user input never flows into the path. `Storage::disk('local')` is root-locked to `storage/app/private/`, so even a malicious DB row cannot escape the storage root. No path traversal vector. |
| T-17-07 | Denial of Service | Unauthenticated hot-looping the download route | accept | Route sits behind `auth` + `verified` + `role:*` middleware — unauthenticated requests 302 to /login before any file I/O. Rate limiting is Laravel framework default; no incident-specific quota needed (PDFs are < 5 MB, cheap to stream). |
| T-17-08 | Spoofing | Session hijack → download another cell's report | transfer | Out of scope for this gap-closure. Session security is the framework's responsibility (Fortify + 2FA, Phase 1). Mitigation exists at a lower layer; this route inherits it. |
| T-17-09 | Tampering | User manipulates `{incident}` URL param to enumerate UUIDs | mitigate | Incident PKs are UUIDv4 (192-bit entropy) — enumeration-resistant. Route binds via implicit Eloquent resolution; invalid UUID returns 404 before Gate runs. Gate further restricts access. |
