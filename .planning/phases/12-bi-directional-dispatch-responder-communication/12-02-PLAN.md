<objective>
Build the dispatch-side messaging UI: a collapsible Messages section in the IncidentDetailPanel, unread message tracking via useDispatchFeed, notification indicators (queue badge, topbar MSGS count), and subtle audio cue for incoming messages.

Purpose: Dispatchers currently have no visibility into responder messages. This plan gives dispatchers a complete messaging interface to view, send, and track messages per incident, with notification indicators ensuring no message goes unnoticed.

Output: DispatchMessagesSection component, updated useDispatchFeed with MessageSent listener and unread Map, message notification audio in useAlertSystem, unread badges on QueueCard and DispatchTopbar, auto-expand logic in Console.vue.
</objective>

       - Add function `clearUnread(incidentId: string)`: deletes entry from `unreadByIncident` Map
       - Add function `getMessages(incidentId: string): DispatchMessageItem[]`: returns messages from Map or empty array
       - Add function `addLocalMessage(incidentId: string, message: DispatchMessageItem)`: pushes to messagesByIncident for optimistic local push on send
       - In the existing `IncidentStatusChanged` handler, when removing RESOLVED incidents (`splice`), also delete from `unreadByIncident` and `messagesByIncident` Maps
       - In the state sync handler (`onStateSync`), clear both Maps to reset on reconnect
       - Return: `{ tickerEvents, unreadByIncident, totalUnreadMessages, messagesByIncident, clearUnread, getMessages, addLocalMessage }`
