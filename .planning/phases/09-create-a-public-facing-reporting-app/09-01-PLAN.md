       ```php
       Route::prefix('v1/citizen')->group(function () {
           Route::get('incident-types', [CitizenReportController::class, 'incidentTypes'])
               ->middleware('throttle:citizen-reads')
               ->name('api.citizen.incident-types');
           Route::get('barangays', [CitizenReportController::class, 'barangays'])
               ->middleware('throttle:citizen-reads')
               ->name('api.citizen.barangays');
           Route::post('reports', [CitizenReportController::class, 'store'])
               ->middleware('throttle:citizen-reports')
               ->name('api.citizen.reports.store');
           Route::get('reports/{token}', [CitizenReportController::class, 'show'])
               ->middleware('throttle:citizen-reads')
               ->name('api.citizen.reports.show');
       });
       ```

  <action>
    1. Create StoreCitizenReportRequest (use `php artisan make:request Api/V1/StoreCitizenReportRequest --no-interaction`):
       - authorize() returns true (public endpoint)
       - Rules (array-style per project convention):
         - 'incident_type_id' => ['required', 'exists:incident_types,id']
         - 'description' => ['required', 'string', 'min:10', 'max:1000']
         - 'caller_contact' => ['required', 'string', 'max:20'] (per locked decision: required)
         - 'caller_name' => ['nullable', 'string', 'max:100']
         - 'location_text' => ['nullable', 'string', 'max:500']
         - 'latitude' => ['nullable', 'numeric', 'between:-90,90']
         - 'longitude' => ['nullable', 'numeric', 'between:-180,180']
         - 'barangay_id' => ['nullable', 'exists:barangays,id'] (fallback when GPS denied)

    2. Create CitizenIncidentTypeResource:
       - Returns: id, name, category, code, default_priority, description

    3. Create CitizenReportResource:
       - Maps internal status to citizen-facing label using CITIZEN_STATUS_MAP constant:
         PENDING -> 'Received', TRIAGED -> 'Verified',
         DISPATCHED/ACKNOWLEDGED/EN_ROUTE/ON_SCENE/RESOLVING -> 'Dispatched',
         RESOLVED -> 'Resolved'
       - Returns: tracking_token, type (incident type name), category, priority, status (mapped), barangay (name), location_text, description, submitted_at (created_at formatted)
       - NEVER exposes incident_no, internal UUID, or internal status enum

    4. Create CitizenBarangayResource:
       - Returns: id, name only (no boundary geometry)

    5. Create CitizenReportController (use `php artisan make:controller Api/V1/CitizenReportController --no-interaction`):
       - Constructor injects BarangayLookupService
       - incidentTypes(): Query IncidentType::active()->public()->orderBy('sort_order'), return CitizenIncidentTypeResource::collection()
       - barangays(): Query Barangay::orderBy('name')->select('id', 'name'), return CitizenBarangayResource::collection()
       - store(StoreCitizenReportRequest):
         a. Look up incident type, get default_priority
         b. Build data array: incident_type_id, priority (from type default), status=PENDING, channel=App, caller_name, caller_contact, location_text, notes=description
         c. Generate tracking_token via Incident::generateTrackingToken()
         d. If latitude+longitude provided: create Point::makeGeodetic, run BarangayLookupService::findByCoordinates
         e. Else if barangay_id provided: use that directly
         f. Create incident
         g. Create IncidentTimeline entry (event_type='incident_created', event_data={source: 'citizen_app', tracking_token: token})
         h. Load incidentType + barangay relations
         i. Dispatch IncidentCreated event
         j. Return CitizenReportResource with 201 status
       - show(string $token): Find incident by tracking_token or 404, load relations, return CitizenReportResource

    6. Update admin incident type management:
       - Add 'show_in_public_app' to StoreIncidentTypeRequest and UpdateIncidentTypeRequest rules arrays
       - Add show_in_public_app boolean toggle to IncidentTypeForm.vue (checkbox or switch near is_active)
       - Pass show_in_public_app in form data from the admin form

       - Test all behaviors listed above
