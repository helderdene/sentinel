  <action>
    1. Create `resources/js/composables/usePushSubscription.ts`:
       - Export `usePushSubscription()` composable
       - Reactive refs: `isSubscribed`, `isSupported` (check `'serviceWorker' in navigator && 'PushManager' in window`)
       - `VAPID_PUBLIC_KEY` from `import.meta.env.VITE_VAPID_PUBLIC_KEY`
       - `subscribe()` async function:
         a. Get service worker registration via `navigator.serviceWorker.ready`
         b. Call `registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(VAPID_PUBLIC_KEY) })`
         c. POST subscription JSON (endpoint, keys.p256dh, keys.auth, content_encoding 'aesgcm') to `/push-subscriptions` with CSRF token from `document.querySelector<HTMLMetaElement>('meta[name="csrf-token"]')?.content`
         d. Set `isSubscribed.value = true`
         e. Store `'push-subscribed'` flag in localStorage
       - `unsubscribe()` async function:
         a. Get current subscription via `registration.pushManager.getSubscription()`
         b. Call `subscription.unsubscribe()`
         c. DELETE to `/push-subscriptions` with endpoint
         d. Set `isSubscribed.value = false`
         e. Remove localStorage flag
       - `checkExistingSubscription()` async function:
         a. If push not supported, return
         b. Get registration, get subscription
         c. If subscription exists, set `isSubscribed.value = true`
       - Helper `urlBase64ToUint8Array(base64String: string): Uint8Array`
       - Return `{ isSubscribed, isSupported, subscribe, unsubscribe, checkExistingSubscription }`

    2. Create `resources/js/components/PushPermissionPrompt.vue`:
       - Uses `usePushSubscription` composable
       - Uses `usePage` from Inertia to check if user is authenticated (`page.props.auth?.user`)
       - Only renders for authenticated users
       - Shows a custom dialog/banner (NOT triggering the browser permission prompt directly) asking "Enable push notifications?" with explanation text:
         - For responders: "Get notified of new assignments even when IRMS is closed"
         - For dispatchers: "Get notified of critical P1 incidents even when IRMS is closed"
         - For other roles: "Get notified of critical events even when IRMS is closed"
       - Two buttons: "Enable Notifications" (calls subscribe()) and "Not Now" (dismisses, sets localStorage 'push-prompt-dismissed' flag)
       - Show condition: user is authenticated AND push is supported AND not already subscribed AND no 'push-prompt-dismissed' in localStorage AND no 'push-subscribed' in localStorage
       - On mount, call `checkExistingSubscription()` to detect existing subscription
       - Style: fixed overlay with centered card, design system tokens (bg-t-surface, text-t-text, border-t-border, shadow-4), backdrop blur
       - Use Transition for fade-in animation

    3. Update `resources/js/app.ts`:
       - Import PushPermissionPrompt
       - Add to the render array alongside existing ReloadPrompt:
         ```typescript
         render: () => [h(App, props), h(ReloadPrompt), h(PushPermissionPrompt)],
         ```
       - Also add CSRF meta tag to `app.blade.php` if not already present (check first -- it may already be there from Inertia). The composable needs `meta[name="csrf-token"]` -- verify this exists in the Blade template. If missing, add `<meta name="csrf-token" content="{{ csrf_token() }}">` to the `<head>`.

    4. Run `npm run build` to verify no TypeScript or build errors.
    5. Run `npm run lint && npm run format` for frontend files.
