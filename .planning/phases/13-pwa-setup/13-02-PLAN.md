<objective>
Wire Web Push notifications into the IRMS application so that responders receive push alerts for new assignments and ack timeouts, and dispatchers receive push alerts for P1 incidents -- all functioning when the app is backgrounded or closed. Includes backend notification infrastructure (laravel-notification-channels/webpush), push subscription management endpoints, event listeners hooked into existing broadcast events, a delayed queue job for ack timeout detection, and a frontend permission prompt with subscription management composable.

Purpose: Critical events (assignments, P1 alerts, ack timeouts) reach users even when the app is not in the foreground, which is essential for emergency response operations where seconds matter.
Output: Working Web Push notification pipeline from event trigger through server-side dispatch to service worker display.
</objective>
