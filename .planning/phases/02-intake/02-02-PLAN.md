    2. **PrioritySelector.vue** -- Component accepting `modelValue` (P1-P4 string), `suggestion` ({priority, confidence} or null), and emitting `update:modelValue`. Renders 4 buttons in a row: P1 (red), P2 (orange), P3 (amber), P4 (green). Active button has filled background; others are outlined. When suggestion is provided, show confidence percentage next to the suggested priority. Use Tailwind for button colors matching IncidentPriority.color() values. P1=red-500, P2=orange-500, P3=amber-500, P4=green-500.

    3. **usePrioritySuggestion.ts** -- Composable that accepts reactive refs for `incidentTypeId` (number|null) and `notes` (string). When incidentTypeId changes or notes change (debounced 500ms), fetch from `/api/priority/suggest?incident_type_id=N&notes=...`. Return reactive `{suggestion: Ref<PrioritySuggestion|null>, isLoading: Ref<boolean>}`. Use AbortController pattern to cancel stale requests. Implement manual debounce (do NOT add VueUse as dependency -- project doesn't have it). Simple ~10 line debounce utility using setTimeout/clearTimeout.

    4. **useGeocodingSearch.ts** -- Composable that accepts a reactive `query` ref (string). When query length >= 3 characters (debounced 300ms), fetch from `/api/geocoding/search?query=...`. Return reactive `{results: Ref<GeocodingResult[]>, isLoading: Ref<boolean>}`. Same debounce/AbortController pattern as usePrioritySuggestion.

    5. **Create.vue** -- The triage form page at `/incidents/create`. Use AppLayout. Section order per locked decision: Channel + Caller Info, Incident Details (type + priority), Location, Notes.
       - **Channel + Caller Info section:** Channel as Select dropdown (5 channels from props). Caller name and caller contact as Input fields.
       - **Incident Details section:** Incident type as grouped Combobox (types grouped by category from props). PrioritySelector component with auto-suggestion from usePrioritySuggestion composable. When incident type is selected, auto-fill priority from its default_priority and trigger suggestion. When notes change, re-trigger suggestion (500ms debounce).
       - **Location section:** Location text Input with autocomplete dropdown showing geocoding results from useGeocodingSearch. Selecting a result populates hidden latitude/longitude fields and triggers barangay lookup display. Show "Barangay: [name]" as read-only field that updates when coordinates are set (barangay is resolved server-side on submit, but show the barangay name from prop data if coordinates match).
       - **Notes section:** Textarea for free-text notes.
       - **Form submission:** Use `useForm` + Wayfinder `store()` action from IncidentController. Submit button: "+ Create Incident".
       - Show form validation errors per field. Show processing state on submit button.
       - Include breadcrumbs: Dashboard > Incident Queue > New Incident.
