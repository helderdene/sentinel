    1. Create useIntakeFeed.ts composable:
       - Accept initial pendingIncidents and triagedIncidents arrays as parameters (from Inertia props)
       - Maintain reactive local copies (like Queue.vue WebSocket pattern from Phase 3)
       - Subscribe to dispatch.incidents channel via useEcho:
         - IncidentCreated: Add new incident to feed with `fresh: true` flag for slideIn animation. Dedup by ID. Only add if status is PENDING.
         - IncidentStatusChanged: If new_status is TRIAGED, move incident from pending to triaged list. If it was the active selection, clear selection.
       - Expose: feedIncidents (computed, filtered by active tab), pendingCount, triagedCount, activeFilter (ref: 'all'|'pending'|'triaged'), setFilter, selectIncident, activeIncident (ref), channelCounts (computed from pending incidents)
       - Feed capped at 100 incidents (oldest removed when exceeded)
       - Triaged incidents have opacity flag for rendering at 55%

    2. Create useIntakeSession.ts composable:
       - Track per-session metrics in memory (no backend persistence):
         - received: ref(0) -- incremented when IncidentCreated received
         - triaged: ref(0) -- incremented when triage form submitted
         - handleTimes: ref<number[]>([]) -- time between card selection and triage submission
       - Expose: received, triaged, pending (computed: received - triaged), avgHandleTime (computed: mean of handleTimes in seconds, 0 if empty), recordReceived(), recordTriaged(handleTimeMs)
       - Resets on page load (in-memory = resets on login since operators go directly to intake)
  </action>
