    1. **useDispatchMap.ts** composable -- the core map engine:
       - Accept `containerId: string` parameter
       - Initialize MapLibre map on `onMounted` with: container, CartoCDN Dark Matter style (`https://basemaps.cartocdn.com/gl/dark-matter-gl-style/style.json`), center [125.5406, 8.9475] (Butuan City), zoom 13, maxPitch 0, dragRotate false (2D only per locked decision)
       - On `map.on('load')`: add 3 GeoJSON sources: 'incidents' (promoteId: 'id'), 'units' (promoteId: 'id'), 'connections'
       - **Incident layers** (4 layers on 'incidents' source):
         - `incident-halo`: circle, radius 18, color by priority (P1 #dc2626, P2 #ea580c, P3 #ca8a04, P4 #16a34a), opacity 0.15, blur 1
         - `incident-pulse`: circle, radius 14, color by priority, opacity 0.08 (subtle pulse ring)
         - `incident-border`: circle, radius 8, color white, opacity 0.8
         - `incident-core`: circle, radius 6, color by priority, stroke-width 0, opacity 1
       - **Unit layers** (3 layers on 'units' source):
         - `unit-glow`: circle, radius 14, color by status (AVAILABLE #16a34a, DISPATCHED #2563eb, EN_ROUTE #2563eb, ON_SCENE #ca8a04, OFFLINE #6b7280), opacity 0.15, blur 1
         - `unit-border`: circle, radius 7, color white, opacity 0.9
         - `unit-body`: circle, radius 5, color by status, opacity 1
       - **Connection lines** layer on 'connections' source: line type, color by priority, width 2, dasharray [2, 4]
       - **Click handlers**: incident click (emit selected incident ID), unit click (emit selected unit ID), empty map click (emit deselect). Use `map.on('click', 'incident-core', ...)` for layer-specific clicks.
       - Expose functions:
         - `setIncidentData(incidents: DispatchIncident[])`: Convert to GeoJSON FeatureCollection and call setData on 'incidents' source. Each feature: id = incident.id, geometry from coordinates, properties: {id, priority, status, incident_no}.
         - `setUnitData(units: DispatchUnit[])`: Same pattern for units. Properties: {id, status, callsign}.
         - `updateUnitPosition(unitId: string, lng: number, lat: number)`: Use updateData() for partial GPS update. Store current positions in a Map<string, [number, number]> for animation.
         - `animateUnitTo(unitId: string, toLng: number, toLat: number, durationMs?: number)`: requestAnimationFrame interpolation from current position to new position with ease-out cubic (see RESEARCH.md Pattern 5). Default 1000ms.
         - `updateConnectionLines(assignments: Array<{incident: DispatchIncident, unit: DispatchUnit}>)`: Build LineString features from unit coordinates to incident coordinates. Set 'connections' source data.
         - `flyToIncident(incident: DispatchIncident)`: map.flyTo with center on incident coordinates, zoom 15, duration 1000ms.
         - `flyToUnit(unit: DispatchUnit)`: map.flyTo with center on unit coordinates.
         - `switchStyle(dark: boolean)`: Use `map.setStyle()` with Dark Matter or Positron URL. Listen for `style.load` to re-add all sources/layers. Store current GeoJSON data in composable state for re-application.
       - Cleanup: `onUnmounted` calls `map.remove()`, cleans up animation frames
       - Return: `{ map, isLoaded, setIncidentData, setUnitData, updateUnitPosition, animateUnitTo, updateConnectionLines, flyToIncident, flyToUnit, switchStyle, onIncidentClick, onUnitClick, onDeselect }`
       - Event callbacks as composable-level registries: `onIncidentClick(cb)`, `onUnitClick(cb)`, `onDeselect(cb)`

    2. **useAlertSystem.ts** composable -- extend existing audio infrastructure:
       - Reuse the existing AudioContext initialization pattern from useWebSocket.ts (initAudio with click/keydown unlock)
       - Define PRIORITY_TONES record per CONTEXT.md:
         - P1 CRITICAL: frequencies [880, 660, 880, 660, 880, 660], durations [0.25, 0.25, 0.25, 0.25, 0.25, 0.25] (3 alternating pulses, 1.5s)
         - P2 HIGH: frequencies [700, 700], durations [0.3, 0.3] (2 pulses, 0.6s)
         - P3 MEDIUM: frequencies [550], durations [0.3] (1 pulse, 0.3s)
         - P4 LOW: frequencies [440], durations [0.2] with gentle decay (soft chime)
       - `playPriorityTone(priority: IncidentPriority)`: Create oscillator chain per RESEARCH.md Pattern 4. Gain at 0.3, exponential ramp to 0.01.
       - `playAckExpiredTone()`: Distinct warning tone (e.g., 2 rapid 600Hz pulses)
       - `triggerP1Flash()`: Add CSS class to document.body that applies an inset box-shadow pulse animation (red glow, 3 pulses over ~2s). Use CSS animation class that auto-removes after animationend event. Non-blocking -- map and panels stay interactive.
       - Add the P1 flash CSS animation to app.css:
         ```css
         @keyframes p1-flash { 0%,100% { box-shadow: inset 0 0 0 0 transparent; } 50% { box-shadow: inset 0 0 40px 10px rgba(220,38,38,0.3); } }
         .p1-flash-active { animation: p1-flash 0.7s ease-in-out 3; }
         ```
       - Return: `{ playPriorityTone, playAckExpiredTone, triggerP1Flash }`
