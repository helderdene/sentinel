<objective>
WebSocket real-time wiring for the dispatch console (all broadcast events consumed), mutual aid modal, and visual verification checkpoint. This plan makes the dispatch console fully live.

Purpose: Without WebSocket listeners, the dispatch console is a static snapshot. This plan wires IncidentCreated, IncidentStatusChanged, UnitLocationUpdated, UnitStatusChanged, and MutualAidRequested events to mutate local state, update the map, trigger alerts, and update metrics in real-time. Mutual aid completes the dispatch workflow.
Output: useDispatchFeed composable, MutualAidModal, all WebSocket listeners wired, live dispatch console ready for visual verification.

  <action>
    1. **useDispatchFeed.ts** composable -- the real-time event hub:
       - Accept: localIncidents (Ref of DispatchIncident[]), localUnits (Ref of DispatchUnit[]), mapRef (from useDispatchMap), alertSystem (from useAlertSystem)
       - Subscribe to `dispatch.incidents` channel:
         - `IncidentCreated`: Add to localIncidents in priority-sorted position (using the findIndex + splice pattern from Phase 3). Dedup by ID. Call setIncidentData to update map markers. Play playPriorityTone(e.priority). If P1: also call triggerP1Flash(). Add to ticker events.
         - `IncidentStatusChanged`: Find incident in localIncidents, update status. If new_status is RESOLVED, remove from list. Update map markers. Add to ticker events.
         - `MutualAidRequested`: Add to ticker events. Optionally add to selected incident timeline if it matches.
       - Subscribe to `dispatch.units` channel:
         - `UnitLocationUpdated`: Find unit in localUnits, call animateUnitTo with new coordinates (smooth transition). Update unit coordinates in localUnits. Update connection lines.
         - `UnitStatusChanged`: Find unit in localUnits, update status. Update map unit markers. If status changed to AVAILABLE, recalculate nearby units for selected incident.
       - Wire `onStateSync` from useWebSocket: Replace localIncidents and localUnits with fresh data from StateSyncResponse. Re-render all map data.
       - Maintain `tickerEvents: Ref of TickerEvent[]` -- last 20 events for live ticker display
       - Return: `{ tickerEvents }`

    2. **MutualAidModal.vue**: Triggered by "REQUEST MUTUAL AID" button in IncidentDetailPanel:
       - Accept: incident (DispatchIncident), agencies (DispatchAgency[]), open (boolean)
       - Suggested agencies: filter agencies where their incident_types include the current incident's incident_type_id. Highlight with star icon.
       - All 5 agencies always shown. Suggested ones first with highlight, others below.
       - Each agency card: name, code badge, contact_phone (clickable tel: link), contact_email, radio_channel
       - Free-text notes textarea
       - Select agency (radio or click to highlight), type notes
       - "REQUEST AID" button: POST to dispatch.mutual-aid via Wayfinder action with {agency_id, notes}. Close modal on success. Show success toast.
       - "CANCEL" button to close
       - Use Reka UI Dialog for modal (following existing pattern from intake/admin)

    3. **Wire MutualAidModal into IncidentDetailPanel.vue**:
       - Add `showMutualAid` ref
       - "REQUEST MUTUAL AID" button sets showMutualAid = true
       - Mount MutualAidModal with selected incident and agencies prop

    4. **Wire live ticker into DispatchTopbar.vue**:
       - Accept tickerEvents as prop (or via inject from layout)
       - Display last 3-5 events as scrolling/cycling text in the ticker area
       - Each event: "[incident_no] [priority] [type] at [location]" (truncated)
       - Auto-scroll new events in, old ones out. Use CSS animation or simple interval cycling.

    5. **Wire everything in Console.vue**:
       - Create local reactive copies of incidents and units from Inertia props (following Phase 3 pattern: `const localIncidents = ref([...props.incidents])`)
       - Initialize useDispatchFeed with localIncidents, localUnits, mapRef, alertSystem
       - Initialize useWebSocket for connection state and state-sync
       - Pass tickerEvents to DispatchTopbar
       - Pass localIncidents to DispatchQueuePanel (not raw props -- local copies get WebSocket mutations)
       - Pass localUnits to UnitStatusPanel
       - On incident data change (watch): update map incident data, rebuild connection lines
       - On unit data change (watch): update map unit data
       - Ensure assignment actions refresh local data (after POST succeeds, either local mutation or router.reload for the affected incident)

    Run `npm run lint` and `npm run format`.
