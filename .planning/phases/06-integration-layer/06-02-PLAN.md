       ```php
       interface WeatherServiceInterface
       {
           /**
            * Get current weather advisories for the operational area.
            *
            * @return array{advisories: array<int, array{type: string, level: string, title: string, description: string, affected_barangays: string[], issued_at: string, expires_at: string}>, current: array{rainfall_mm_hr: float, wind_speed_kph: float, wind_direction: string, temperature_c: float}}
            */
           public function getCurrentAdvisories(): array;

           /**
            * Get current weather conditions for a specific location.
            *
            * @return array{rainfall_mm_hr: float, wind_speed_kph: float, wind_direction: string, temperature_c: float}
            */
           public function getCurrentConditions(float $latitude, float $longitude): array;
       }
       ```

    2. Create `app/Services/StubPagasaWeatherService.php`:
       - Implements WeatherServiceInterface
       - `getCurrentAdvisories()`: Returns static PAGASA-style data with REAL Butuan City barangay names (Libertad, Baan Km 3, Baan Riverside, Limaha, Doongan, Langihan, Obrero, Villa Kananga) and realistic advisory text ("Rainfall Warning Level 2 -- Butuan City", "Wind Advisory -- Agusan del Norte")
       - Advisory types: rainfall, wind, flood
       - Advisory levels: yellow, orange, red (PAGASA 3-level system)
       - `getCurrentConditions()`: Returns static Butuan City weather (tropical: ~28C, high humidity, moderate rainfall)
       - Both methods log via `Log::info('StubPagasaWeatherService::methodName', [params])`

    3. Create `app/Contracts/HospitalEhrServiceInterface.php`:
       ```php
       interface HospitalEhrServiceInterface
       {
           /**
            * Send patient pre-notification to hospital EHR system (HL7 FHIR R4).
            *
            * @param array{blood_pressure?: string, heart_rate?: int, spo2?: int, gcs?: int} $vitals
            * @param string[] $assessmentTags
            * @return array{status: string, reference_id: string, fhir_payload: array}
            */
           public function preNotify(
               string $hospitalId,
               string $incidentNo,
               string $incidentType,
               array $vitals,
               array $assessmentTags,
               string $etaMinutes,
               string $unitCallsign,
           ): array;
       }
       ```

    4. Create `app/Services/StubHospitalEhrService.php`:
       - Implements HospitalEhrServiceInterface
       - `preNotify()`:
         a. Looks up hospital name from `config('hospitals')` array by hospitalId
         b. Builds FHIR R4 Patient resource (resourceType: "Patient", identifier with incident number, name: "Emergency Patient")
         c. Builds FHIR R4 Encounter resource (resourceType: "Encounter", status: "planned", class: { code: "EMER" }, subject ref to Patient, serviceProvider with hospital name, reasonCode from incident type, period.start)
         d. Builds FHIR R4 Observation resources for each non-null vital (BP, HR, SpO2, GCS) with proper LOINC codes
         e. Wraps in FHIR Bundle (resourceType: "Bundle", type: "transaction", entry array)
         f. Logs full payload via `Log::info('StubHospitalEhrService::preNotify', [hospitalId, incidentNo, payload summary])`
         g. Returns `['status' => 'accepted', 'reference_id' => 'FHIR-STUB-' . strtoupper(substr(md5($incidentNo), 0, 8)), 'fhir_payload' => $bundle]`

    5. Update `app/Providers/AppServiceProvider.php` register():
       - Add: `$this->app->bind(WeatherServiceInterface::class, StubPagasaWeatherService::class);`
       - Add: `$this->app->bind(HospitalEhrServiceInterface::class, StubHospitalEhrService::class);`
