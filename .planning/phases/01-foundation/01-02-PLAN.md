    **Step 1: Create routes/admin.php**

    Create `routes/admin.php` with a route group wrapped in `['auth', 'verified', 'role:admin']` middleware, prefix `admin`, and name prefix `admin.`:
    - `Route::resource('users', AdminUserController::class)` -- full CRUD
    - `Route::resource('incident-types', AdminIncidentTypeController::class)` -- full CRUD
    - `Route::resource('barangays', AdminBarangayController::class)->only(['index', 'edit', 'update'])` -- read + edit only, no create/delete (barangays are seeded, admin can only edit metadata)

    Register `routes/admin.php` in `bootstrap/app.php` by adding it to the `withRouting()` call. Use the `then:` parameter to load the admin routes file:
    ```php
    ->withRouting(
        web: __DIR__.'/../routes/web.php',
        commands: __DIR__.'/../routes/console.php',
        health: '/up',
        then: function () {
            Route::middleware(['web', 'auth', 'verified', 'role:admin'])
                ->prefix('admin')
                ->name('admin.')
                ->group(base_path('routes/admin.php'));
        },
    )
    ```

    **Step 2: Create 3 controllers using `php artisan make:controller`**

    `AdminUserController`:
    - `index()`: Query all users with their unit relationship (eager loaded). Return `Inertia::render('admin/Users', ['users' => $users, 'roles' => UserRole::cases(), 'units' => Unit::all()])`.
    - `create()`: Return `Inertia::render('admin/UserForm', ['roles' => UserRole::cases(), 'units' => Unit::all()])`.
    - `store(StoreUserRequest)`: Create user from validated data. Hash password. If role is Responder and unit_id provided, associate. Redirect to admin.users.index with success flash.
    - `edit(User)`: Return `Inertia::render('admin/UserForm', ['user' => $user->load('unit'), 'roles' => ..., 'units' => ...])`.
    - `update(UpdateUserRequest, User)`: Update user. If password provided, hash it; otherwise exclude from update. Redirect back with success flash.
    - `destroy(User)`: Soft-check (don't delete yourself). Delete user. Redirect with success flash.

    `AdminIncidentTypeController`:
    - `index()`: Query all incident types ordered by category then sort_order. Group by category for the frontend. Return `Inertia::render('admin/IncidentTypes', ['types' => $types, 'categories' => $categories])`.
    - `create()`: Return `Inertia::render('admin/IncidentTypeForm', ['priorities' => IncidentPriority::cases(), 'categories' => existing categories])`.
    - `store(StoreIncidentTypeRequest)`: Create type from validated data. Redirect to admin.incident-types.index.
    - `edit(IncidentType)`: Return `Inertia::render('admin/IncidentTypeForm', ['type' => $type, ...])`.
    - `update(UpdateIncidentTypeRequest, IncidentType)`: Update type. Redirect with success.
    - `destroy(IncidentType)`: Soft-disable instead of hard delete (set is_active=false). Redirect.

    `AdminBarangayController`:
    - `index()`: Query all barangays ordered by district, name. Return `Inertia::render('admin/Barangays', ['barangays' => $barangays])`. Exclude the boundary geography column from the query for performance (only need name, district, population, risk_level for the list).
    - `edit(Barangay)`: Return form with current metadata (no boundary data sent to frontend).
    - `update(UpdateBarangayRequest, Barangay)`: Update only risk_level, population, district. Redirect.

    **Step 3: Create 5 FormRequest classes using `php artisan make:request`**

    All use array-style rules per existing codebase convention:

    `StoreUserRequest`: name required|string|max:255, email required|email|unique:users, password required|min:8|confirmed, role required|in:admin,dispatcher,responder,supervisor, unit_id nullable|exists:units,id (required_if:role,responder), badge_number nullable|string|max:50, phone nullable|string|max:30.

    `UpdateUserRequest`: Same as Store but email unique:users,{user_id}, password nullable (optional on update), role required.

    `StoreIncidentTypeRequest`: category required|string|max:50, name required|string|max:100, code required|string|max:20|unique:incident_types, default_priority required|in:P1,P2,P3,P4, description nullable|string, is_active boolean, sort_order nullable|integer.

    `UpdateIncidentTypeRequest`: Same as Store but code unique:incident_types,{type_id}.

    `UpdateBarangayRequest`: district nullable|string|max:50, population nullable|integer|min:0, risk_level nullable|in:low,moderate,high,very_high. NO boundary field accepted.

    **Step 4: Run Wayfinder generation**

    After creating all controllers and routes, run `php artisan wayfinder:generate --no-interaction` to generate TypeScript route functions for the new admin controllers. This is required so that Task 2's Vue pages can import Wayfinder actions from `@/actions/Admin/AdminUserController` etc.

    **Step 5: Create tests**

    Create 3 test files using `php artisan make:test --pest`:
    - `tests/Feature/Admin/AdminUserTest.php` -- test CRUD, role enforcement, validation
    - `tests/Feature/Admin/AdminIncidentTypeTest.php` -- test CRUD, category grouping, disable
    - `tests/Feature/Admin/AdminBarangayTest.php` -- test index, edit, update metadata only

    Each test uses factories from Plan 01 with `->admin()` state for the acting user, and verifies non-admin users get 403.

    Run `vendor/bin/pint --dirty --format agent` after all PHP changes.
